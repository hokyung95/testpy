# DuckDB & PugSQL 연동 문제 조치 내역 (2026-07-14)

본 문서는 DuckDB와 PugSQL을 연동하는 과정에서 발생한 에러 현상, 원인 분석 및 해결 조치 내용을 정리한 작업 내역서입니다.

## 1. 개요
* **목적**: PugSQL 라이브러리를 통해 DuckDB 인메모리 데이터베이스(`:memory:`)에 연결하여 DDL(테이블 생성), DML(데이터 삽입), DQL(데이터 조회) 쿼리를 성공적으로 연동

## 2. 발생한 에러 및 원인 분석

### ① `NoSuchModuleError: Can't load plugin: sqlalchemy.dialects:duckdb`
* **현상**: `queries.connect('duckdb:///my_data.db')` 호출 시 dialect 플러그인을 찾을 수 없다는 에러 발생
* **원인**: PugSQL은 내부적으로 SQLAlchemy를 활용하여 DB에 연결합니다. SQLAlchemy가 DuckDB와 통신하려면 전용 Dialect 어댑터인 `duckdb-engine` 패키지가 환경에 설치되어 있어야 하지만, 기존 가상 환경에는 `duckdb` 코어 패키지만 존재하고 어댑터가 누락되어 있었습니다.

### ② `ArgumentError: Expected string or URL object, got <_duckdb.DuckDBPyConnection ...>`
* **현상**: `queries.connect(conn)`와 같이 `duckdb.connect()`로 받아온 raw 커넥션 객체를 직접 전달하였을 때 에러 발생
* **원인**: PugSQL의 `connect()` 인터페이스는 SQLAlchemy의 `create_engine()` 호출 규격을 따르므로, Python DB-API 2.0용 raw 커넥션 객체가 아닌 SQLAlchemy 커넥션 스트링(URL 문자열) 형식인 `duckdb:///:memory:` 형태로 입력을 받아야 합니다.

### ③ `ParserException: Parser Error: syntax error at or near ":"`
* **현상**: 데이터 삽입 쿼리(`queries.insert_user(...)`) 실행 시 SQL 구문 오류 발생
* **원인**: `schema.sql` 쿼리 정의 파일 내부에서 네임드 파라미터 바인딩을 위한 `:` 기호 뒤에 공백이 존재하였습니다. (예: `(: id,: name,: email)`) SQL 파서가 이를 정상적인 바인딩 이름으로 인식하지 못해 에러가 발생했습니다.

### ④ `InvalidInputException: Invalid Input Error: No open result set`
* **현상**: DDL(테이블 생성) 및 데이터 삽입 후 최종 사용자 조회를 수행할 때 조회 결과셋을 정상적으로 읽어오지 못하는 오류 발생
* **원인**: 인메모리 데이터베이스(`:memory:`) 특성상 매 커넥션 획득 시마다 초기화되거나 상태 유실이 발생할 수 있습니다. 각 쿼리가 별도의 연결 컨텍스트에서 수행되는 것을 방지하고 단일 세션 내에서 트랜잭션을 유지해야 합니다.

## 3. 해결 조치 사항

### ① 어댑터 패키지 설치
* 가상 환경에 SQLAlchemy용 DuckDB dialect 연동 모듈을 추가 설치하였습니다.
  ```bash
  pip install duckdb-engine
  ```

### ② 소스 코드 수정 ([pugsql_test.py](file:///d:/pyprj/testpy/pugsql_test.py))
* `duckdb.connect()`를 사용해 커넥션을 생성하여 전달하는 부분을 제거하고, `duckdb:///:memory:` 커넥션 스트링을 넘겨주도록 변경하였습니다.
* 인메모리 데이터의 정합성과 커넥션 유지를 위해 DDL, DML, DQL 전 과정을 하나의 `queries.transaction()` 블록으로 감싸주었습니다.

```python
# 수정 후 핵심 코드
queries.connect('duckdb:///:memory:')

with queries.transaction():
    queries.create_users_table()
    queries.insert_user(id=1, name='김철수', email='chulsoo@example.com')
    queries.insert_user(id=2, name='이영희', email='younghee@example.com')
    
    users = queries.get_all_users()
    for u in users:
        print(u)
```

### ③ SQL 쿼리 파일 수정 ([schema.sql](file:///d:/pyprj/testpy/resource/sql/schema.sql))
* 바인딩 변수의 구문 오류를 해결하기 위해 콜론 뒤에 있던 공백을 모두 제거하였습니다.
```sql
-- 수정 전
VALUES (: id,: name,: email);

-- 수정 후
VALUES (:id, :name, :email);
```
